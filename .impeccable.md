## Design Context

### Users
Developers using Claude Code and Codex CLI in terminal-first workflows. They need a compact status line that surfaces model, repo, context usage, and rate-limit state without breaking focus during coding sessions.

### Brand Personality
Restrained, professional, technical. The interface should feel calm and reliable rather than flashy, with enough hierarchy to be scannable in a dense terminal environment.

### Aesthetic Direction
Preserve the current terminal HUD language and stay dark-mode first. Use subdued neutrals, one clear accent, and theme-tinted supporting text instead of decorative effects or loud contrast jumps.

### Design Principles
1. Keep the status line dense but immediately scannable.
2. Use semantic color roles so emphasis and supporting text stay consistent across themes.
3. Preserve the current terminal-native aesthetic; refine it rather than restyling it.
4. Favor accessible contrast for informative text, especially reset times, placeholders, and unavailable states.
5. Keep Bash, PowerShell, and Codex rendering behavior visually aligned.
